# Phase 4: CHANGELOG & repo hygiene

## Overview

Make the repo look professionally maintained AND honestly disclosed: Keep-a-Changelog
`CHANGELOG.md` (the authoritative v1.0.0 release-notes source), README badges + Install
section with case/proxy caveats, `SECURITY.md` (unsigned-binary disclosure), `CONTRIBUTING.md`
(pinned GoReleaser version + Homebrew prose TODO), GitHub issue/PR templates, roadmap note.
Verification = render/link/format checks.

## Requirements

- Functional:
  - `CHANGELOG.md` — Keep a Changelog format; `[1.0.0] - 2026-05-18` summarizing the 10
    shipped phases + M1 fix (from `docs/project-roadmap.md` + git log); `[Unreleased]` =
    M2 backlog. Compare/tag links. **This is the canonical release-notes text** consumed by
    Phase 5 (extracted to `.github/release-notes.md` for GoReleaser `--release-notes`).
  - `.goreleaser.yaml` archive `files:` — **add `CHANGELOG.md`** to the existing
    `["README.md","LICENSE"]` list (now that the file exists; Phase 2 deliberately omitted it).
  - `README.md` — badge row (CI, latest release, Go version, license, pkg.go.dev);
    `## Installation` with: `go install github.com/bavanchun/Typeburn@latest` in a code block
    + an explicit note **"module path is case-sensitive — the capital `T` is required"** +
    **"`go install` may lag the Go module proxy by up to ~1h after a release; downloading
    the release binary is immediate."**; download-binary path; Homebrew "planned, not yet
    available". `typeburn --version` usage line.
  - `SECURITY.md` — responsible-disclosure contact + explicit statement: **release binaries
    are NOT cryptographically signed; integrity is provided only by HTTPS transport and
    `checksums.txt` generated by the same pipeline; verify checksums and trust the build
    pipeline accordingly.** (Honest accepted-risk disclosure; signing is intentionally out
    of v1 scope.)
  - `CONTRIBUTING.md` — prerequisites (Go 1.26; **exact pinned GoReleaser version
    `v2.x.y`** — same as Phase 2/3, stated explicitly); `make build/test/test-race/lint/fmt`;
    conventional commits (no AI refs); PR flow; how to test a release via `make snapshot`
    (build/archive only) vs the disposable-tag publish dry-run; **Homebrew enablement TODO
    as prose** (provision `bavanchun/homebrew-tap`, mint a *fine-grained* PAT scoped to ONLY
    that repo `Contents: write` with expiry — never a classic/org-wide PAT — store as repo
    secret, add a `homebrew_casks:` block to `.goreleaser.yaml`).
  - `.github/ISSUE_TEMPLATE/bug_report.yml` (asks for `typeburn --version` output, OS,
    terminal, repro, expected/actual), `feature_request.yml`, `config.yml`
    (`blank_issues_enabled: false`), `.github/PULL_REQUEST_TEMPLATE.md` (checklist: tests
    pass, gofmt, conventional commit, CHANGELOG updated).
  - `docs/project-roadmap.md` — annotate v1.0 section: "Tagged `v1.0.0` (2026-05-18), see
    GitHub Releases" (minimal edit; history not rewritten).
- Non-functional: shields.io standard URLs; no broken relative links.
